# HR Analytics Data Pipeline using Apache Spark
### MSc Computing (Data Analytics) — Dublin City University | 2025

## Overview
A production-style HR analytics data pipeline built with Apache Spark 
(PySpark) that integrates heterogeneous HR and Payroll datasets into 
unified, analytics-ready outputs.

The pipeline processes ~3 million records resulting from complex 
many-to-many joins and produces structured analytical tables similar 
to those generated by enterprise HR platforms such as Workday HCM, 
BambooHR, and Sage People.

Unlike proprietary systems, this solution is fully transparent, 
reproducible, and cloud-ready.

---

## Business Problem
In most organisations:
- HR and Payroll systems operate in separate silos
- Schemas and formats are incompatible
- No shared employee identifiers exist

This fragmentation prevents unified analysis of:
- Demographics vs compensation
- Engagement vs turnover
- Workforce structure across departments

This pipeline solves that problem using distributed data processing 
and domain-based schema harmonisation.

---

## Why Apache Spark?
Spark was selected over MapReduce, Hive, and Storm because it provides:
- In-memory distributed processing for large-scale analytics
- A unified API (DataFrames, SQL, functional transformations)
- Efficient handling of joins and aggregations
- Seamless scalability from local development to cloud clusters
- Strong industry adoption in modern data engineering

## Pipeline Architecture

### ETL Flow

Raw CSV Files (HR + Payroll)
↓
Load into Apache Spark (Distributed DataFrame Reader)
↓
Data Cleaning & Standardisation
↓
Feature Engineering (Age, Tenure, Pay Grade)
↓
Schema Harmonisation (DeptCategory Mapping)
↓
Many-to-Many Join (~3M rows)
↓
Aggregated Analytical Tables
↓
CSV Export (Ready for BI Tools)

---

## Key Technical Features
- Schema harmonisation using department-level domain mapping
- Intelligent handling of business-meaningful null values
- Date parsing and normalisation across inconsistent formats
- Feature engineering (age, tenure, pay grade categories)
- Distributed many-to-many joins producing 3M+ rows
- Aggregation strategies to control join explosion
- Pure Spark implementation (no Pandas or local processing)

---

## Analytical Outputs
The pipeline generates five production-ready analytical tables:

| Output File | Description |
|---|---|
| `demographics.csv` | Workforce demographics (age, gender, race) |
| `engagement.csv` | Department-level engagement metrics |
| `compensation.csv` | Compensation summaries (base, overtime, total) |
| `turnover.csv` | Turnover rate analysis |
| `training.csv` | Training outcomes assessment |

All outputs are in the `Outputs/` folder and are ready for 
Tableau, Power BI, or Excel.
